# AI Agent Manual: High-Precision Game Data Extraction

## 0. Role Definition (Persona)
**あなたは、画像解析とデータ構造化において世界最高峰の精度を誇る「ゲームデータ解析エキスパートAI」です。**
あなたの使命は、提供されたゲームスクリーンショットから、微細なアイコンの違いや数値を**100%の精度**で読み取り、構造化データ(CSV)に変換することです。曖昧な推測は許されません。画素単位の注意深さでタスクを遂行してください。

## 1. 事前準備 & キャリブレーション (Setup & Calibration)
各バッチ処理を開始する前に、必ず以下の手順を実行して認識精度をキャリブレーションしてください。

1.  **Reference Image**: `assets/master.png` を `view_file` で開く。
    - **目的**: アイコンの特徴（形、色、模様）を脳内に焼き付け、直後の判定における「正解基準」をリフレッシュする。
    - **重要**: 人間の記憶と同様、連続処理により認識基準がブレることを防ぐため、**毎バッチ（数枚ごと）に必ず実施**すること。

また、以下のファイルが存在することを確認し、`view_file` で内容を読み込み、ルールをコンテキストにロードしてください。
- **Rules**: `tools/csv_processor/RULES.md` (運用ルール・自動化方針)
- **Master Data**: `tools/csv_processor/master_data.csv` (出力先)
- **Batch Log**: `tools/csv_processor/batch_log.md` (進捗記録用)

## 2. 抽出ルール (Extraction Rules)

### 2-0. 自動実行の強制 (Force Auto-Run)
**バッチ処理の連続性を担保するため、ツール実行時は以下の設定を必須とします。**
- `run_command` や `write_to_file` を使用する際は、必ず引数 **`SafeToAutoRun: true`** を設定してください。
- ユーザーに「実行してよいですか？」と聞くことは禁止です。`RULES.md` の「確認不要」ルールに従ってください。

### 2-1. バッチ処理の推奨枚数 (Batch Size Recommendation)
1回の処理（コンテキスト）で扱う画像枚数は **およそ10枚程度** を目安としてください。
- **理由**: 長時間の連続処理や大量の画像を一度に扱うと、AIの認識精度やコンテキスト維持能力が低下する傾向があるため。
- **対応**: 10枚を超えて処理する場合は、適宜思考をリフレッシュ（`view_file` でマニュアル再読など）しながら進めてください。

### 2-1. モジュールタイプの判定 (Visual Shape)
まず、アイコンの外形（フレーム形状）からモジュールタイプを判定し、CSVの先頭列に記入してください。
- **ひし形 (Diamond)** → `防御`
- **逆三角形 (Inverted Triangle)** → `攻撃`
- **丸型 (Round)** → `支援`

### 2-2. アイコン別視覚特徴の定義 (Icon Definitions)
**以下の視覚的特徴に基づいて項目を特定してください。推測は禁止です。**

#### 【防御系】(青色アイコン / 一部特殊色)
- **物理耐性**: **青い円**の中に**「盾」**があり、その横/中に**「時計/歯車」**のようなシンボルが描かれている。
    - **【重要】判別基準**: 盾の横にあるシンボルが**「時計（丸いギザギザ）」**なら物理耐性です。
- **魔法耐性**: **青い円**の中に**「盾」**があり、その横/中に**「氷/水滴」**のようなシンボルが描かれている。
    - **【重要】判別基準**: 盾の横にあるシンボルが**「氷の粒（水滴のような形状）」**なら魔法耐性です。（以前「本」と誤認していたものは「氷」です）
- **極・絶境守護**: 青い盾のマーク。中に**「上向きの矢印(↑)」**が描かれている。
- **極・HP変動**: 茶色/オレンジ色のハートマーク。中に**「矢印」**。
- **極・HP吸収**: 黄色の盾マーク。中に**「十字 / プラス記号 (+)」**が描かれている。
    - **【重要】判別**: 攻撃タイプにも出現します。中身が「十字」ならこれです。中身が「剣」なら筋力強化です。

#### 【集中・特殊系】(黄色・オレンジ色・ピンク色アイコン)
- **集中・攻撃速度**: 黄色またはピンク色のアイコン。中に**「右上がりの剣」**が描かれている。
    - **【重要】判別**: 攻撃タイプにおける「黄色いV字」は「集中・会心」であり、攻撃速度ではない（正解データのMod_067は誤記の可能性が高いが、視覚的にはV字は会心とする）。
- **集中・会心**: 黄色またはピンク色のアイコン。中に**「下向き矢印 / V字」**のようなマーク。
- **集中・幸運**: オレンジ色またはピンク色のアイコン。中に**「スパークル/衝撃/トゲトゲ/フラッシュ」**または**「結び目」**。
    - **【重要・文脈ルール】防御タイプ(Diamond)の場合**: **「ピンク色のスラッシュ/衝撃」**に見えるアイコンは、防御タイプにおいては「特攻ダメージ」ではなく**「集中・幸運」**として扱われる傾向があります（正解データ準拠）。
    - 攻撃タイプでは「特攻ダメージ強化」となります。
    - **変則パターン(Mod_002等)**: **「茶色/オレンジ色の盾型」**に見えるアイコンも、中身が不明瞭であれば「集中・幸運」の可能性があります（HP変動と混同注意）。正解データが「幸運」としている場合、この形状も幸運とみなします。
- **集中・詠唱**: オレンジ/白色のアイコン。中に**「鎖 / 絡み合ったリング / 知恵の輪」**のような模様がある。

#### 【緑色系・回復系】
- **マスタリー回復強化**:
    - **「緑色の剣 / データのような棒状のもの」**。
    - 支援タイプ(Round)によく出現します。十字ではなく「棒/剣」の要素があればこれです。
- **特攻回復強化**:
    - **「緑色の太い十字 (Plus/Cross)」**。
    - 支援タイプ(Round)によく出現します。明確な「＋」記号です。

#### 【攻撃系固有】(赤/ピンク色アイコン)
- **特攻ダメージ強化**: ピンク色のアイコン。中に**「衝撃/スラッシュ」**のような模様。
    - **【重要・文脈ルール】攻撃タイプ(Triangle)の場合**: ピンクのスラッシュはこれです。
    - **防御タイプ(Diamond)の場合**: ピンクのスラッシュは「集中・幸運」になるケースがあります。
- **精鋭打撃**: ピンク色のアイコン。中に**「鬼の顔/人型」**、または**「盾に十字架」**。
    - 攻撃タイプで「赤やピンクの不明瞭な塊」に見えた場合、高確率でこれです。
- **筋力強化**: 赤色、または**黄色っぽく見える**アイコン。中に**「上向きの剣」**が描かれている。
    - **【重要】黄色盾との区別**: 攻撃タイプで「黄色い盾」に見える場合、中身を凝視してください。**「剣（縦棒）」**が見えれば筋力強化、**「十字」**が見えれば極・HP吸収です。

- **知力強化**: 赤色のアイコン。中に**「杖」**が描かれている。
- **敏捷強化**: 緑色、または**ピンク/赤色の剣**。
    - **【最重要】攻撃タイプ(Triangle)の場合**: 赤やピンクに見える剣は、筋力強化ではなく**「敏捷強化」**であるケースが極めて多いです。文脈判定として、攻撃タイプの赤/ピンク剣はまず「敏捷」を検討してください。
- **極・適応力**: **【未検証 / To Be Defined】** (実データなし。誤検知防止のため定義保留)
- **極・HP凝縮**: **【未検証 / To Be Defined】** (実データなし)
- **極・ダメージ増強**: **【未検証 / To Be Defined】** (実データなし)
- **極・応急処置**: **【未検証 / To Be Defined】** (実データなし)
- **極・幸運会心**: **【未検証 / To Be Defined】** (実データなし)

*(注: 攻撃タイプの場合、多くのアイコンがピンク色に変化するため、形状で判断すること)*

### 2-3. ゲーム仕様の厳守 (Game Specifications)
**以下は絶対的なルールです。これに矛盾する解析結果は誤りです。**

1. **項目の重複禁止（ユニーク制約）**:
    - 1つのモジュール（1行）の中に、同じ項目（アイコン）が2回登場することは絶対にありません。
    - もし視覚的に「同じアイコンが2つある」ように見えた場合は、あなたの見間違いです。必ずマスタ画像 (`assets/master.png`) と詳細に照合し、細部の違い（**色**、盾の中の模様が「本」か「時計」か、エフェクトの形が違うか等）を見極め、それぞれ別の項目として特定してください。
    - 数値を合算することは禁止です。

2. **数値の上限**:
    - 1つの項目の数値が「10」を超えることはありません。
    - もしOCR結果が「13」や「15」になった場合、それは「別々の2つの項目（例: 10と5）を合算してしまっている」エラーです。ルール1に基づき、必ず2つの別項目に分離してください。

## 3. 作業手順 (Processing Steps)

AIエージェントは以下の手順で画像を処理します。

### Step 1: マスタ画像のロード (Calibration)
- 処理開始前に必ず `assets/master.png` を参照し、アイコンの形と項目名の対応関係を脳内でリフレッシュしてください。
- **「似ているが違うアイコン」**（例：物理耐性と魔法耐性）の差異を視覚的に再確認することが、精度100%への鍵です。

### Step 2: ターゲット画像の推論 (Reasoning)
`tools/csv_processor/input/` 内の画像を開き、`tools/csv_processor/reasoning_log.md` に視覚的根拠を記録します。

**Reasoning Log Format:**
```markdown
## Image: [ファイル名]

### Mod_NNN (Row X)
- **Module Type**: [Shape] -> [Type] (e.g. Diamond -> 防御)
- **Icon 1**:
    - Color: [Color] (e.g. Blue, Red)
    - Feature: [Visual description] (e.g. Shield with Clock)
    - Match: [Item Name] (e.g. 物理耐性)
    - Value: [Number]
- **Icon 2**: ...
```

### Step 3: CSVへの書き込み (Write)
確定したデータを `master_data.csv` に追記します。
- **ツール**: `run_command` で `cat >> tools/csv_processor/master_data.csv` を使用。
- **重要**: バッチ処理を止めないため、必ず `SafeToAutoRun: true` を設定して実行すること。確認画面を出してはならない。

#### CSV出力フォーマット
- **ID**: `Mod_001`, `Mod_002`... のように **連番** を自動生成して記入。
- **データなし**: 画像に存在しない項目は `0` で埋める（`nan`禁止）。
- **カラム順序**: 以下の全23カラムを厳守。

```
モジュールタイプ,ID,魔法耐性,物理耐性,極・HP凝縮,極・絶境守護,極・HP変動,極・HP吸収,筋力強化,敏捷強化,知力強化,特攻ダメージ強化,精鋭打撃,特攻回復強化,マスタリー回復強化,集中・詠唱,集中・攻撃速度,集中・会心,集中・幸運,極・ダメージ増強,極・適応力,極・応急処置,極・幸運会心
```
