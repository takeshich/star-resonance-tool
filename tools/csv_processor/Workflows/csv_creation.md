description: tools/csv_processor/input/ 内のすべての画像に対してCSV作成バッチ処理を実行します。
---
1. `tools/csv_processor/AI_AGENT_MANUAL.md` を `view_file` で読み込み、すべての「必須ルール」と「抽出手順」を遵守することを宣言してください。
// turbo
2. `tools/csv_processor/assets/master.png` を表示し、アイコン判定の視覚的基準をロードしてください。
// turbo
3. `tools/csv_processor/input/` フォルダ内の画像を `list_dir` で一覧取得してください。
// turbo
4. `input/` の画像一覧から、`batch_log.md` を参照して「既に処理済みの画像」を除外してください。
// turbo
5. 未処理のインプット画像に対して、以下の手順を繰り返してください：
    **【絶対ルール】AIのコンテキスト破綻を防ぐため、今回5. 以下の**「画像1枚ごとの処理ループ（最大5〜10回繰り返し）」**を実行する。絶対にまとめて処理・出力してはならない。
    a. 未処理の画像を**1枚**開く。
    b. マスタ画像 (`assets/master.png`) および定義と視覚的に照合し、モジュールタイプとアイコン情報（能力値）を解析する。
    c. その画像1枚分の解析詳細根拠をファイルに分割して追記する（`list_dir` 等で既存ログを確認し、上書きしないよう一意な連番や日時を付与する。例: `reasoning_log_batch2.md`）。
    d. 解析が完了した1枚分のデータを追加行として `master_data.csv` に出力する。(IDは `Mod_NNN` の連番)
    e. 処理が完了したファイル名を `batch_log.md` に追記して記録する。
    f. 次の画像へ移る。（上限の5〜10枚に達したらこのバッチ処理を終了する）
6. もし画像が上限に達して処理を停止した場合は、ユーザーに対して**「コンテキスト保護のため処理を一時停止しました。新しいチャットを立ち上げて、再度 /csv_creation を実行してください」**と報告してタスクを完全に終了してください。
